## 结合 AI 生成表结构

```markdown
帮我生成一个 mysql 合理通用的 xxx 表，基本的一些要求如下：

1. sql 的关键字都用大写，增强可读性；
2. 索引，主键，唯一键对字段的约束，和表结构定义放在一起，尽量避免使用`CREAT`来创建；
3. 表需要有`created_at`，`updated_at`，`deleted_at`字段，类型是时间戳；
4. 使用`utf8mb4`编码格式；
5. 表，字段的注释要清晰，易于理解；
6. 需要有自增主键，主键类型为`BIGINT`；
7. 表不存在才创建，避免删除后重建；
8. 表的`COLLATE`使用`utf8mb4_general_ci`；
9. 根据你的经验，给一些字段合理的默认值，请尽量避免默认值是 NULL 的情况；
10. 字段名，表名使用反引号，防止意外情况；
11. 只需要给出表结构定义的 SQL 语句， 不需要其他额外的 SQL 操作；
12. 如果字段涉及到价格，使用`INT`类型，单位是分；
13. 表名在可能的情况下使用复数;
14. 不需要外键约束；
15. 默认引擎为 InnoDB；
16. 根据你的经验，给字段添加合理的索引，主键，唯一键等能加速查询的约束。
```
